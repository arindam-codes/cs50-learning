While working toward the **Credit** problem, I experimented with strings and
indexing to understand how credit card numbers are processed in C.

Key insights gained:
- Credit card numbers are best handled as strings, not integers
- Iterating digit-by-digit requires correct indexing and bounds handling
- Characters must be converted to digits (`'0'` â†’ `0`)
- Processing digits from right to left is a design decision
- Many operations rely on understanding memory layout rather than memorizing rules

Because of this preparation, the official walkthrough and lecture explanations
felt intuitive with very low cognitive load.
